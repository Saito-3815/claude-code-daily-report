# 営業日報システム 要件定義書

## システム概要

営業担当者が日々の活動を報告し、上長がフィードバックを行うための日報管理システム

## 使用技術

- **言語**: TypeScript
- **フレームワーク**: Next.js (App Router)
- **UIコンポーネント**: shadcn/ui + Tailwind CSS
- **APIスキーマ定義**: OpenAPI (Zodによる検証)
- **DBスキーマ定義**: Prisma.js
- **テスト**: Vitest
- **デプロイ**: Google Cloud Run

## 機能要件

### 1. 日報管理

- 営業担当者は1日1件の日報を作成できる
- 日報には報告日、作成者情報を含む

### 2. 訪問記録

- 1つの日報に対して複数の訪問記録を登録できる
- 訪問記録には顧客情報と訪問内容を記載する

### 3. Problem/Plan管理

- 日報に現在の課題・相談事項（Problem）を記載できる
- 日報に翌日の予定（Plan）を記載できる

### 4. コメント機能

- 上長はProblem/Planに対してコメントを付けられる

### 5. マスタ管理

- 顧客マスタ：取引先情報を管理
- 営業マスタ：営業担当者情報を管理（上長との関係含む）
